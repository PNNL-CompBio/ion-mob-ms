# Source Scripts - R Processing Modules

## Overview

The `src` directory contains specialized R scripts for metadata extraction, ion mobility drift time processing, and feature annotation. These are core data processing components integrated into the IMDASH pipeline.

## Files

### Phase I: Metadata Extraction

- **`I_Extract-Agilent-d-file-metadata.R`** - Agilent raw data metadata extraction
  - Extracts instrument parameters from Agilent .d files
  - Metadata captured:
    - Acquisition time and duration
    - Mass analyzer settings (capillary voltage, fragmentor voltage)
    - Ion mobility parameters (drift voltage, multiplier voltage)
    - Temperature and pressure conditions
    - Sample information (name, solvent, concentration)
  - Output: Structured metadata CSV or JSON
  - Integration: Called by DM_cli.py and preprocessing modules
  - Special handling for Agilent's proprietary data format

### Phase II: Drift Time Processing

- **`II_Parse-DT-as-RT-mzML.R`** - Drift time interpretation and mzML parsing
  - Converts raw drift time (DT) measurements into retention time (RT) equivalent
  - mzML parsing and validation:
    - Reads mzML files generated by ProteoWizard
    - Validates data structure and required fields
    - Handles multiple scan types and fragmentation patterns
  - Drift time to retention time mapping:
    - Creates virtual retention time from drift coordinates
    - Enables compatibility with tools expecting RT dimension
    - Handles 2D and 3D ion mobility separations (Single Field, SLIM, Stepped Field)
  - Output: Reformatted mzML with DT→RT conversion
  - Integration: Critical preprocessing step for MZmine compatibility

### Phase V: Feature Annotation

- **`V_Annotate-calibrated-features.R`** - Ion mobility feature annotation
  - Annotates detected features with calibrated ion mobility properties
  - CCS calculation integration:
    - Reads AutoCCS output with calculated CCS values
    - Matches features to CCS determinations
    - Resolves ambiguities when multiple ions match single CCS
  - Annotation additions:
    - CCS values to feature intensity matrix
    - Confidence scores for CCS assignments
    - Ion identity suggestions based on CCS matching
    - Cross-references to compound databases
  - Output: Feature matrix with full ion mobility annotation
  - Integration: Final post-processing step before statistical analysis

## Data Structures

### Input Data Format

Raw ion mobility data typically:
- Agilent .d directory (proprietary format)
- Multiple spectra with retention time and drift time dimensions
- Variable number of parallel ion pathways

### Intermediate Processing

Conversion pathway:
```
Agilent .d file
    ↓
Extract Metadata (Script I)
    ↓ (input metadata)
ProteoWizard mzML
    ↓
Parse DT as RT (Script II)
    ↓ (reformatted mzML)
MZmine Feature Detection
    ↓
Feature CSV
    ↓
AutoCCS CCS Calculation
    ↓
Annotate Features (Script V)
    ↓
Final Annotated Features CSV
```

### Output Data Format

Annotated features CSV:
```
mz,rt,ccs,intensity,annotation,confidence,ion_id,database_match
123.456,10.5,120.5,50000,MATCH,0.95,M+H,Leucine-Isoleucine
234.567,12.3,145.2,30000,CANDIDATE,0.78,2M-H,Lysophospholipid
```

## Usage

### Individual Script Execution

```bash
# Extract metadata from Agilent file
Rscript I_Extract-Agilent-d-file-metadata.R \
        input_directory=/path/to/file.d \
        output_file=metadata.csv

# Parse drift time and generate mzML variant
Rscript II_Parse-DT-as-RT-mzML.R \
        input_mzml=/path/to/input.mzML \
        output_mzml=/path/to/output_with_rt.mzML

# Annotate features with CCS information
Rscript V_Annotate-calibrated-features.R \
        features_csv=/path/to/features.csv \
        ccs_results=/path/to/autoccs_out.txt \
        output=/path/to/annotated_features.csv
```

### Integration with Pipeline

When running full IMDASH pipeline:

1. **Data Preparation**: Script I extracts metadata (informational)
2. **Format Conversion**: ProteoWizard converts to mzML
3. **Preprocessing**: Script II reformats with drift time handling
4. **Feature Detection**: MZmine generates feature list
5. **Ion Mobility Annotation**: AutoCCS calculates CCS
6. **Final Annotation**: Script V combines all information

### Docker Integration

Within Docker containers, scripts invoked via:

```bash
docker run -v /data:/work imdash:latest Rscript /scripts/I_Extract-*.R ...
```

## Technical Details

### Metadata Extraction (Script I)

Extracts from Agilent .d files:
- Binary format parsing (platform-dependent)
- Requires Agilent data access libraries
- Recovers both method parameters and acquisition conditions
- Handles multiple spectrum types in single run

### Drift Time Processing (Script II)

DT to RT conversion logic:
- Drift time: Separation within ion mobility dimension (milliseconds)
- Retention time: Time in liquid chromatography (minutes)
- Mapping: DT value → virtual RT for tools expecting single time dimension
- Preservation: Original DT values maintained in annotations

**Supported Workflow Types**:
- **Single Field**: Constant voltage, single DT dimension
- **SLIM**: Extended path, higher DT resolution
- **Stepped Field**: Multiple voltages, complex DT relationships

### Feature Annotation (Script V)

Matching algorithm:
1. Read calculated CCS values from AutoCCS
2. Read detected features and their m/z values
3. Query CCS database for m/z matches
4. Calculate confidence scores based on:
   - Error in m/z matching
   - CCS precision
   - Database match likelihood
5. Generate annotated output with all metrics

## Common Issues

### Issue: "Cannot read Agilent .d file"
- **Cause**: Missing Agilent data access libraries
- **Solution**: Ensure Docker image includes Agilent libraries
- **Alternative**: Pre-extract metadata on Windows machine with Agilent software

### Issue: "DT dimension not found in mzML"
- **Cause**: Pre-existing mzML file doesn't contain ion mobility data
- **Solution**: Ensure input mzML comes from Agilent ProteoWizard (preserves DT)
- **Check**: `strings input.mzML | grep -i "ionMobility"`

### Issue: "Feature annotation too slow"
- **Cause**: Large feature count or slow database queries
- **Solution**: Reduce feature list to significant peaks only
- **Alternative**: Pre-calculate CCS database locally

### Issue: "CCS values not assigned to features"
- **Cause**: m/z matching too strict or database mismatch
- **Solution**: Check CCS output format matches expected structure
- **Debug**: Manually inspect first 10 features in both files
